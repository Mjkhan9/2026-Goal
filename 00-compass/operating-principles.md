# Operating Principles

The non-negotiable rules that govern how work is done in this initiative. Update only when a principle genuinely changes — not for tactical shifts.

## Core principles

1. **Sell outcomes, not AI.** Buyers purchase business results. AI is the leverage, not the product.
2. **Files over chats.** Any insight, decision, or artifact worth keeping lives in this repo, not in a chat window.
3. **Judgment is the scarce resource.** AI compresses labor; the operator owns direction, quality, and trust.
4. **Fixed-scope, repeatable, high-ROI.** Prefer offers that can be priced, delivered, and improved like a product.
5. **Prove before systemizing.** Do it manually with real buyers before turning anything into a system or product.
6. **One active bet at a time.** Focus beats optionality during the validation phase.
7. **Cheap to kill, fast to ship.** Validate with the smallest honest version; discard quickly if signal is weak.
8. **No fabrication.** Case studies, metrics, and claims must be real.

## How to use this file

If a decision contradicts a principle here, either the decision is wrong or the principle needs to be updated explicitly with a decision record in `01-state/decisions/`.
