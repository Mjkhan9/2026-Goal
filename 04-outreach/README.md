# 04 — Outreach

Prospecting and sales. The goal is to move qualified buyers through a visible pipeline.

## Files & subfolders

- `pipeline.md` — single live view of prospects and their stage.
- `targets/` — ICPs and prospect lists (one file per segment or source).
- `messages/` — outreach templates, cold-email sequences, follow-ups, scripts.
- `proposals/` — proposal templates and per-prospect proposal drafts.

## Pipeline stages (default)

1. `identified` — known to exist, matches ICP
2. `contacted` — first outreach sent
3. `replied` — engaged in conversation
4. `qualified` — confirmed fit, budget, decision-maker
5. `proposal` — proposal sent
6. `won` | `lost` | `nurture`

## Rules

- Never store personal contact info or client-identifying details in plaintext if the repo is ever shared beyond private access. Prefer initials or codenames for sensitive pipeline entries.
- Each sent message is worth saving as a template if it was effective.
