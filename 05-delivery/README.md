# 05 — Delivery

How the work actually gets done. This folder grows as real engagements reveal what should be systemized.

## Subfolders

- `sops/` — standard operating procedures. One clearly-named procedure per file.
- `playbooks/` — end-to-end playbooks for a full engagement type.
- `checklists/` — short, tactical checklists used during execution.
- `qa/` — quality control: review checklists, acceptance criteria, common failure modes.

## Rules

- Write SOPs only **after** a process has been done successfully at least once. No speculative SOPs.
- Each SOP includes: purpose, when to use it, inputs, steps, outputs, failure modes.
- Promote recurring sub-steps into their own reusable SOP.
