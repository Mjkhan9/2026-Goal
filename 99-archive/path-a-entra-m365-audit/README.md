# Archive — Path A: Entra / M365 Identity Hygiene Audit for SMBs

**Status:** shelved as of 2026-04-17
**Decision:** `01-state/decisions/0002-pivot-to-research-first-diagnostic-model.md`

## What this was
The first offer hypothesis developed for this initiative: a fixed-scope
audit of Microsoft Entra / M365 identity and access configuration for
small and mid-sized businesses. The hypothesis was that SMBs accumulate
unmanaged identity complexity and would pay for a bounded, expert-delivered
cleanup and documentation engagement.

## Why it was shelved
The project shifted to a research-first, diagnostic-led model before this
hypothesis was tested with buyers. The Entra/M365 angle was anchored on
operator expertise rather than observed buyer demand. A hypothesis shaped by
supply (what the operator can do) rather than demand (what Houston SMBs
actually need and will pay for) is not the right starting point.

No discovery conversations were conducted. No buyer signal was collected.
The hypothesis was never invalidated — it was set aside before testing.

## Potential reuse
If diagnostic work on Houston-area SMBs reveals that identity and access
problems are common, prioritized by buyers, and willingly paid for, this
path can be revived. The files and thinking here are preserved for that
possibility.
