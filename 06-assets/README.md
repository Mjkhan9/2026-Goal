# 06 — Assets

Reusable, evergreen artifacts. The library that compounds over time.

## Subfolders

- `templates/` — document, email, proposal, brief, and report templates.
- `case-studies/` — real engagement write-ups. **Never fabricate.** Include context, problem, approach, outcome, and (where possible) measured result.
- `proof/` — raw proof material (screenshots, metrics, testimonials, links) that can be cited in outreach, proposals, and case studies.
- `brand/` — voice, positioning, naming, taglines, bio variants.

## Rules

- Prefer improving an existing template over creating a new one.
- Case studies require permission before publishing externally; mark each one with a `permission:` status.
- Keep binary proof files reasonably sized; link to originals when bulky.
